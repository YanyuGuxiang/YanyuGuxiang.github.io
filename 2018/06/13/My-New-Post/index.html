Hello World

发表于 2018-06-13 | 分类于 品味生活 | 阅读次数:
字数统计: 1,364 | 阅读时长 ≈ 5

  人生是一场旅行。我们经历了几次轮回，才换来这个旅行。而这个旅程很短，因此不妨大胆一些，去爱一个人，去攀一座山，去追一个梦……有很多事我都不明白。但我相信一件事。上天让我们来到这个世上，就是为了让我们创造奇迹。

前言

  喜欢写博客的人会经历三个阶段。

  • 第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
  • 第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
  • 第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

引自阮一峰

  今天！算是有个自己的小窝了，这博客折腾了我一个月，不过这期间还是蛮高兴的，看着自己的博客越来越是那么回事，这改改那改改。终于变成了自己喜欢的模样。心生欢喜。仅此一篇，记录一下心情…

偶遇

  喜欢写博客的经历三个阶段我觉得挺有道理的，只不过我第一阶段和第二阶段都没经历过。直接到了第三个阶段… 虽然都说搞技术的人有一个自己的技术博客是多好的一件事。我自己也有几年的开发时间了，之前也接触过很多的Blog，但是确实没有要自己写博客的这个概念。

  直到有一天。

  我不知怎的就进入了强哥的博客，笔名叫做 - 纯洁的微笑，先传过去：www.ityouknow.com 刚一进博客真的让我眼前一亮，简洁、大方。而且强哥的文采也很棒，把所有的文章都读了一遍，特别是一些品味人生的故事，也能给我很多的思考。就在这么一刻，我突然就迸发出了想要自己搞一个博客的念头，把一些技术的分享和生活的境遇也写在自己的博客里，我突然觉得能有一个自己的博客是多好的一件事… 一不做二不休，我的博客创作之旅开始了。

创作

  首先肯定要清楚搭建一个博客的具体步骤：

  • 找一个可以托管的服务器，毋庸置疑，Giuhub Pages。
  • 安装环境：Git 和 Node.js
  • 选择一个静态网页生成的工具，Jekyll 或者 Hexo。
  • 把本地的生成的静态页面推送到 Github
  • 绑定自己的域名

  Github Pages 是面向用户、组织和项目开放的公共静态页面搭建托管服务，站点可以被免费托管在 Github 上，你可以选择使用 Github Pages 默认提供的域名 github.io 或者自定义域名来发布你的站点。Github Pages 支持利用 Jekyll 生成站点，也同样支持纯 HTML 文档，所以说将你的站点托管在 Github Pages 上是一个不错的选择。

  目前市面上有两大静态网页生成工具 Jekyll 和 Hexo。 我最开始只知道个 Jekyll 所以当时也准备用 Jekyll 来搭建我的博客，也挑选了几个比较满意的主题，也在网上搜了搜用 Jekyll 搭建博客的步骤。就在搜索的过程中，我发现了另外一个东西 Hexo ， 也是比较好奇搜索了一下 Hexo 发现也是个静态网页生成器，跟 Jekyll 的功能类似。也随便搜了下用 Hexo 制作的博客。我觉得真的是很幸运。我碰到了用 Next 主题渲染的博客，刚一看到这个主题.. 我就激动的不得了，因为这个主题我实在是太喜欢了，让我想起了许多我走过的黑白古镇，简直就是梦中的模样，话不多说。就开始用 Next 主题搭建我的博客。

  安装 Hexo 感觉还是蛮简单的，Next 主题的中文文档也是讲解的很全面，很快就出来个雏形。然后推送到 Github 上，这样就可以直接用 Github Pages 默认的域名 github.io 直接访问自己的博客了。因为之前也有在阿里云买过域名，还没有到期，所以直接解析，10分钟之后妥妥的访问，至此，我的博客就搭建完毕了。

峥嵘

  有必要说一下 Markdown 的语法，用敲代码的方式去写文章.. 简直很爽。现在的写作方式是 Atom + Markdown + Github + Hexo。其实搭建博客也就两三天就完成了。最耗费时间的是一些个性化的配置，因为我对前端不是很精通，又想修改很多的地方，加上一些特效之类，所以这里用了很多的时间，期间也搜索了很多的教程，效果甚微，我觉得这些教程也只能把你送到桥头，想要过桥，还需深夜里的一杯特浓咖啡。

  接近尾声，发现第一篇博文竟然写了一下午..修修改改，其实想想 Hello world 也不是这么好写的.. 恍惚间，时间回到大学，课堂上，老师在尽心尽力的给我们做演示，然而我们却依然装不好一个环境。可笑，那竟是少年。
